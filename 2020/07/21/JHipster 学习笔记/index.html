JHipster 学习笔记

JHipster是一个生成、开发和部署Spring Boot + Angular / React / Vue Web应用与Spring微服务的开发平台.

一 主要技术栈

前端技术栈

HTML5 + CSS3 + Bootstrap + TypeScript + Angular …

服务端技术栈

Spring Boot + Spring Security + Maven + MySQL …

部署技术栈

Docker + Kubernetes + Heroku + CloudFoundry …

二 该项目的优点

  1. 创建快速、便捷、省时间，按照官方提示在15分钟内能快速学习该框架，并在几分钟内可以快速搭建。
  2. 开源免费，且使用的Spring Boot，Vue等框架是目前的主流。
  3. 对初学者友好，可以快速看到生成的项目结构。
  4. 可以生成API网关，为所有微服务提供HTTP路由提供负载均衡、有质量，安全的服务和API文档。其中，含有自动生成文档，速率限制，访问控制策略。
  5. 项目含有自动创建微服务，生产环境的监控服务。
  6. 有关于前后端分离，Angular，React，Vue，BootStap4定制功能。
  7. 含有自动代码生成器，有使用的模板，以及监控机制，可以进行快速开发。

3 管理配置文件

JHipster带有两个 Spring 配置文件：

  • dev 用于开发：专注于简化开发和提高生产率
  • prod 用于生产：专注于性能和可扩展性

这些配置文件有两种不同的配置：

  • Maven/Gradle 配置文件在构建时使用。例如./mvnw -Pprod package或./gradlew -Pprod bootJar将打包生产应用程序。
  • Spring 配置文件在运行时工作。一些Spring bean的行为会有所不同，具体取决于配置文件。

Spring配置文件由Maven/Gradle设置，因此我们在两种方法之间具有一致性：您将同时在Maven/Gradle和Spring上有一个prod配置文件。

默认情况下，将使用dev配置文件

五 前后端分离

分离前端和API服务器，可以选择仅 JHipster后端或JHipster前端应用程序

前后端分离后，处理HTTP请求，

1.所有API调用都使用/api 前缀

2.从前端调用/提供的静态资源， 而不应由浏览器缓存。

3.对/app（包含前端应用程序）和/content（包含静态内容，如图像和CSS）的调用应在生产中进行缓存，因为这些资产是经过哈希处理的。

4.调用不存在的路由应将请求转发到index.html。这通常在后端通过ClientForwardController处理。单独部署前端时，需要进行额外配置

另外可以使用NGinx来做代理服务器
